# portfolio-sysadmin

Ce dépôt présente un portfolio de documentation technique orientée **administration systèmes / Linux**.
- des **runbooks** (procédures d’exploitation)
- des **labs** réalisés dans un contexte homelab.

Dans chaque lab, une section **INFRA** décrit l’état de l’environnement (topologie, prérequis, versions) **au moment du Lab**.

## Ce que vous trouverez ici

- **Runbooks** : diagnostic → résolution → validation → rollback (approche “exploitation / production”).
- **Labs** : objectifs → étapes → checkpoints → critères de réussite (approche “apprentissage / compétences”).

L’objectif est de démontrer une capacité à **opérer**, **dépanner** et **documenter** de manière claire et reproductible.

## Démarrage rapide

Parcours conseillé :
1. Consultez `runbooks/` pour un runbook court (ex. journald, systemd, SELinux).
2. Consultez `labs/` pour un lab complet (mise en place + vérifications + critères “terminé”).

## Conventions de rédaction

- Les commandes sont prévues pour être copiées/collées.
- Chaque document vise à inclure :
  - une section **Infra** 
  - une section **Validation** (comment vérifier que cela fonctionne),
  - une section **Rollback** (comment revenir en arrière),
  - des **Notes** (pièges fréquents, variantes, points d’attention).

## Environnement (indicatif)

Documentation principalement rédigée et testée sous Linux (RHEL10 / dérivés) dans un contexte homelab.
